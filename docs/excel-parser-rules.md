# 엑셀 파서 규칙 (공연 매출)

## 목적
공연(콘서트/뮤지컬/연극) 판매현황 엑셀을 공통 규칙으로 파싱하여
`PerformanceDailySales`, `PerformanceDailySalesGrade`에 저장할 수 있도록
일관된 파서 규칙을 정의한다.

## 입력 파일/시트 범위
- 입력: 공연 판매현황 엑셀 파일(장르 무관)
- 우선 시트: `회차별판매`
- 보조 시트: `Sales Report`, `Daily`, `판매상세`, `일별 증감`, `초대내역`, `공연회차`
- 현재 구현 범위: `회차별판매` 기반 파싱

## 기본 전처리 규칙
- 병합 셀로 인한 결측치는 `ffill`로 보정한다.
- 완전 빈 행/열은 제거한다.
- 숫자 필드는 콤마 제거 후 정수로 변환한다(불가 시 0 처리).
- 문자열 양쪽 공백을 제거한다.

## 헤더 탐지 규칙 (회차별판매)
1. 헤더 시작 행은 아래 중 하나를 포함하는 행으로 탐지:
   - `No.` 또는 `No` 또는 `회차`
   - `Date` 또는 `공연일`
   - `Day` 또는 `요일`
   - `Time` 또는 `시간`
2. 헤더는 최소 2행(그룹 헤더 + 세부 헤더)로 구성된다.
3. 헤더 탐지 후 데이터 시작 행은 헤더 블록 직후 행으로 설정한다.

## 컬럼 분류 규칙
### 1) 고정 컬럼
- `No.` / `Date` / `Day` / `Time`
- `CAST` 블록 (예: 순희/유령/영수 등)
  - `Time` 이후 첫 예매처 블록 전까지를 캐스트 컬럼으로 본다.

### 2) 가변 컬럼(예매처/등급)
예매처별로 다음 블록이 반복된다:
- 예매처 그룹: `인터파크(입금)`, `코스타티켓(입금)`, `국립극장(입금)`,
  `라이브러리컴퍼니(입금)`, `비지정석(입금)` 등
- 세부 헤더: `R`, `S`, `A`, `휠체어석`, `비지정석`, `TOT`, `금액`

### 3) 합계/요약 컬럼(선택 파싱)
- `유료 계(입금)`, `유료 계(입금+미입금)`, `전체`, `점유율`, `잔여석`,
  `기준객단가`, `회차별객단가`, `할인율`, `BEP`, `목표매출`, `추가 필요 매출`
- 현재는 `notes`에 JSON 형태로 저장 가능(선택)

## 행 필터링 규칙
아래 조건에 해당하는 행은 스킵한다.
- `No.` 값이 숫자가 아니거나 `TOTAL`, `계`, `합계` 포함
- `Date` 파싱 불가

## 파싱 절차 (회차별판매)
1. `No./Date/Day/Time` 기준으로 회차 행 추출
2. 캐스트 컬럼을 `{역할명: 배우명}` 형태로 수집
3. 예매처 블록별로 다음 값 추출
   - `입금` 블록: 등급별 매수, 금액
   - `입금+미입금` 블록이 있으면 등급별 총매수 추출
4. 등급별 미입금 매수는 아래 규칙으로 계산
   - `미입금 = (입금+미입금) - (입금)`
   - 음수 발생 시 0 처리
5. `초대`(무료) 블록이 있으면 등급별 무료 매수로 저장

## 예매처/등급 정규화 규칙
### 예매처 정규화
- `인터파크(입금)` → `인터파크`
- `코스타티켓(입금)` → `코스타티켓`
- `국립극장(입금)` → `국립극장`
- `라이브러리컴퍼니(입금)` → `라이브러리컴퍼니`
- `비지정석(입금)` → `비지정석`

### 등급 정규화
- `R`, `S`, `A` 그대로 사용
- `휠체어석` → `휠체어석`
- `비지정석` → `비지정석`
- `TOT`, `금액`은 등급이 아닌 합계 컬럼으로 분리

## 예외 처리
- `입금+미입금` 블록이 없을 경우 미입금 매수는 0 처리
- `초대` 블록이 없을 경우 무료 매수는 0 처리
- 등급 컬럼이 누락된 경우 해당 등급은 스킵

## 저장 메모(Notes) 권장 포맷
`PerformanceDailySales.notes`에 아래 정보를 JSON 문자열로 저장 가능
```
{
  "round_no": 12,
  "time": "19:30",
  "cast": {"순희": "박지연", "유령": "송원근"},
  "occupancy_rate": 0.132,
  "remaining_seats": {"R": 137, "S": 114}
}
```
